================================================================================
1. DESIGN OVERVIEW
================================================================================

This Design MVP document defines the complete visual design system and
page-by-page layout blueprint for the Maxera Talent website. It is the single
source of truth for all front-end visual decisions.

GOAL: Deliver a modern, corporate, minimal website that builds credibility and
drives leads. Every design decision should reinforce trust and guide users to
convert.


================================================================================
2. DESIGN PRINCIPLES
================================================================================

Principle               | Description                                    | Practical Application
------------------------|------------------------------------------------|----------------------------------------------
Corporate Credibility   | Must feel like a trusted, established firm     | Dark navy palette, clean type, no gimmicks
Conversion-First        | Every page has a clear next action             | Prominent CTAs, minimal distractions
Mobile-First            | Design for small screens first                 | Stack layouts, tap-friendly buttons (44px+)
Visual Hierarchy        | Guide the eye to what matters most             | Size, weight, colour signal importance
Purposeful Whitespace   | Space communicates professionalism             | Generous padding; never cramped
Consistent System       | Reuse components, tokens, and patterns         | Design tokens + shared component library


================================================================================
3. COLOUR SYSTEM
================================================================================

PRIMARY PALETTE:
  Name          | Hex       | Usage
  --------------|-----------|--------------------------------------------------
  Navy          | #0a1628   | Primary brand — header, hero, section backgrounds
  Navy Dark     | #071020   | Hover states on navy, deep overlay backgrounds
  Teal          | #0d9488   | Primary CTAs, accents, active states, highlights
  Teal Dark     | #0f766e   | Button hover, link hover
  Teal Light    | #e6f7f6   | Tag backgrounds, callout backgrounds
  Surface       | #f8fafc   | Card backgrounds, alternating rows, page body
  Slate         | #1e293b   | Section headings on white, dark body text
  Body          | #334155   | Paragraph text, list items
  Muted         | #64748b   | Labels, helper text, secondary copy, placeholders
  Border        | #e2e8f0   | Card borders, input borders, dividers
  White         | #ffffff   | Text on dark backgrounds, button labels

COLOUR USAGE RULES:
  - Never use teal as a background for large areas — accent only
  - All body text on white must meet WCAG AA contrast (4.5:1 minimum)
  - Navy backgrounds always use white or teal text
  - Interactive elements must have visible hover and focus states

4. TYPOGRAPHY SYSTEM
================================================================================

FONTS:
  Display / Heading : Sora       (Google Fonts)
  Body / UI         : DM Sans    (Google Fonts)
  Fallback          : system-ui, -apple-system, sans-serif
  Loading           : font-display: swap (prevent FOIT)

SCALE:
  Role               | Font     | Weight      | Size (Desktop) | Colour
  -------------------|----------|-------------|----------------|------------------
  Display / Hero H1  | Sora     | Bold 700    | 56–72px        | #0a1628 or #fff
  Page Heading H1    | Sora     | Bold 700    | 40–48px        | #0a1628
  Section Heading H2 | Sora     | SemiBold 600| 32–36px        | #0a1628
  Sub-heading H3     | Sora     | SemiBold 600| 24px           | #1e293b
  Body Paragraph     | DM Sans  | Regular 400 | 16–18px        | #334155
  Body Lead          | DM Sans  | Medium 500  | 18–20px        | #334155
  CTA / Button       | Sora     | SemiBold 600| 15–16px        | #ffffff
  Nav Links          | Sora     | Medium 500  | 14–15px        | #ffffff
  Tag / Badge        | Sora     | Bold 700    | 11–12px        | #0d9488
  Form Labels        | DM Sans  | Medium 500  | 13–14px        | #334155
  Caption / Helper   | DM Sans  | Regular 400 | 12–13px        | #64748b

RESPONSIVE SCALING:
  - Use clamp() for hero titles: clamp(2rem, 5vw, 4.5rem)
  - Body text: 16px mobile, 17–18px desktop
  - Line height: 1.6 body | 1.2 display | 1.4 UI elements
  - Letter-spacing: 0.04em on nav links, tags, uppercase labels

5. SPACING & LAYOUT
================================================================================

SPACING TOKENS:
  Token    | Value | Usage
  ---------|-------|-----------------------------------------------
  space-1  | 4px   | Micro gaps within components
  space-2  | 8px   | Icon-to-text gap, tight inner padding
  space-3  | 12px  | Small component gaps
  space-4  | 16px  | Standard inner padding, list gaps
  space-5  | 20px  | Card inner horizontal padding
  space-6  | 24px  | Card padding, section inner gaps
  space-8  | 32px  | Section top/bottom padding (mobile)
  space-12 | 48px  | Section top/bottom padding (tablet)
  space-16 | 64px  | Section top/bottom padding (desktop)
  space-20 | 80px  | Hero vertical padding
  space-24 | 96px  | Hero bottom padding on desktop

GRID SYSTEM:
  - Max content width: 1280px (max-w-7xl in Tailwind)
  - Horizontal padding: 16px mobile / 24px tablet / 32px desktop
  - Grid columns: 1 col mobile, 2 col tablet, 3 col desktop
  - Gutter: 24px between grid items

8. MOTION & ANIMATION GUIDELINES
================================================================================

Animation        | Trigger              | Duration | Easing       | Implementation
-----------------|----------------------|----------|--------------|-------------------------------
Fade up          | Scroll into view     | 600ms    | ease-out     | opacity 0→1, translateY 24→0
Fade in          | Page load            | 500ms    | ease-out     | CSS keyframe
Card hover lift  | Mouse hover          | 200ms    | ease-in-out  | translateY -4px, shadow up
Button hover     | Mouse hover          | 200ms    | ease-in-out  | translateY -1px, bg change
Nav collapse     | Scroll > 20px        | 300ms    | ease-in-out  | class toggle, height transition
Mobile menu      | Hamburger click      | 250ms    | ease-out     | max-height 0→auto or slide
Form success     | Submit success       | 300ms    | ease         | form → success message
Stagger children | Parent enters view   | +100ms   | ease-out     | animation-delay increments

RULES:
  - Animations must never feel slow or block interaction
  - Shorter durations and subtler movements preferred
  - Reserve scroll-triggered animations for section entries only

9. RESPONSIVE BREAKPOINTS
================================================================================

Breakpoint       | Min Width | Key Layout Changes
-----------------|-----------|---------------------------------------------------
Mobile (default) | 320px+    | Single column, stacked sections, hamburger nav
Tablet sm        | 640px+    | 2-column grids begin (services, values)
Tablet md        | 768px+    | Desktop nav appears, 2-col form+info on Contact
Desktop lg       | 1024px+   | 3-column grids, hero text increases
Desktop xl       | 1280px+   | Max content width (max-w-7xl) applied
Large 2xl        | 1536px+   | No changes; max-w-7xl caps layout


================================================================================
10. ACCESSIBILITY REQUIREMENTS
================================================================================

  - All colour combos meet WCAG 2.1 AA contrast (4.5:1 body, 3:1 large text)
  - Focus states: 2px teal ring on ALL interactive elements (never remove outline)
  - All images must have descriptive alt text
  - Form fields must have associated <label> elements (not placeholder-only)
  - Error messages linked to fields via aria-describedby
  - Navigation keyboard-navigable (Tab, Enter, Escape for mobile menu)
  - Skip-to-content link at top of page for screen readers
  - Semantic HTML: nav, main, section, article, header, footer, h1–h3 hierarchy


================================================================================
11. ICONS & IMAGERY
================================================================================

ICON LIBRARY:
  - Primary      : Lucide React (MIT license, clean SVG)
  - Size scale   : 16px inline | 20px nav | 24px card | 32px feature | 48px hero
  - Industry     : Custom or Heroicons filled, teal colour on light backgrounds
  - Process steps: Numbered circles (navy bg, white number) OR Lucide outlines

IMAGERY GUIDELINES:
  - Photography   : Diverse, professional, US workplace settings
  - MVP placeholders: CSS gradient backgrounds with geometric SVG patterns
  - Avoid generic "diverse people smiling" stock photos
  - Hero background: dark overlay (60–70% opacity) + abstract geometric pattern
  - All images via Next.js Image component with width/height and lazy loading
